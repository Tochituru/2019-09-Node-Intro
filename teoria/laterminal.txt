¡Conquistemos la terminal!

[gitbash]

argumentos ==> Parámetros que les pasamos al proceso que ejecutamos en la terminal (en Node)

cd .. ==> voy a la carpeta anterior

ls ==> veo todas las carpetas y los archivos

clear ==> borra todo lo que he escrito en la terminal

cd [carpeta]/ ==> entro a la carpeta que quiero

mkdir [carpeta] ==> crea una carpeta en la ruta donde estoy.

code . ==> Abre una nueva instancia de Visual Code con la carpeta en la que estoy (puede tirar errores, porque depende de Visual Studio Code)

open . ==> Abre la carpeta en el explorador de archivos (finder) (solo mac)
explore . ==> Abre la carpeta en el explorador de archivos (solo windows)

touch [archivo.extensión] ==> Crea archivos
code [archivo.extensión] ==> Abre el archivo en visual code
open [archivo.extensión] ==> Abre el archivo (en el programa determinado, html en el navegador, en Mac)
start [archivo.extensión] ==> Abre el archivo (en el programa determinado, html en el navegador, en Windows)

node [archivo.extensión] ==> ejecuta el archivo (es decir, el proceso)

rm -rf [archivo.extensión] ==> borra el archivo (PARA SIEMPRE, ESTÁ SEGURA)

mv [archivo.extensión] [nuevoNombreArchivo.extensión] ==> cambia el nombre de archivo (primero el viejo después el nuevo)

git clone [url del repo] ==> cómo clonar un repo

git init ==> inicializar el repo en git (en carpeta)

killall node ==> cierra todos los procesos que pueden estar ejecutándose

-g ==> significa global, es como sass y nodemon
sudo ===> con usuario y contraseña 

npx ==> parte de npm (completo)
